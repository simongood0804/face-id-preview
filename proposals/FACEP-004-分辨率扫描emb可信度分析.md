# 提案：分辨率扫描 — Emb 可信度与输入尺寸关系分析

> 提案编号：FACEP-004  
> 创建日期：2026-07-02  
> 状态：草稿

---

## 1. 动机

当前裁剪窗口固定 900×900，但不同人脸尺度下最佳输入分辨率可能不同。需要系统性地扫描不同输入尺寸下检测置信度的变化规律，以确定最优裁剪尺寸。

## 2. 方案

### 2.1 扫描逻辑

在 `FrameProcessor` 中增加分辨率自动切换逻辑：

```
裁剪尺寸列表：[650, 700, 750, 800, 850, 900, 950, 1000, 1050, 1100, 1150, 1200, 1250, 1300]
每档停留帧数：100（约 3.3 秒 @30fps）
完整一轮：14 × 100 = 1400 帧 ≈ 46 秒
循环方式：到达 1300 后回到 650 继续
```

### 2.2 输出日志

每帧输出 `STAT` 日志行，格式：

```
STAT <size> <time_ms> score=<score> face=<faceId> en=<bool>
```

### 2.3 窗口跟踪

- 人脸中心位于裁剪窗口黄金分割点（0.382）
- 切尺寸时以上次人脸位置为基准重定窗口，避免丢失目标
- 扫描模式下禁止回中

### 2.4 Overlay

- 黄色虚线框实时显示当前裁剪区域
- 框的大小跟随实际裁剪尺寸动态变化

## 3. 统计结果

详见 `docs/分辨率扫描报告.md`。

### 关键结论

| 指标 | 值 |
|------|-----|
| 最佳精度分辨率 | 900-1000（950 最优 score=0.837） |
| 推理耗时 | 71-77ms（各分辨率一致，模型内部固定尺寸） |
| 下降临界 | > 1100 置信度开始下降，> 1250 明显下降 |
| 推荐裁剪尺寸 | 900-950 |
