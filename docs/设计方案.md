## 1. 项目概述

基于 **Android 10**，利用 **EvsSDK**（`com.android.car:evs:1.0.7`）控制摄像头，实现 Face ID 实时预览功能。

核心流程：摄像头取流 → 帧数据处理 → Face ID 特征提取 → 人脸画框 → 预览显示。

### 核心约定

- **算法类型**：自研算法，接口尚未定义
- **算法输入**：帧数据（原始 Buffer + 格式 + 宽高）
- **算法输出**：处理后的帧数据（带人脸画框）+ Face ID 特征向量/标识
- **关键要求**：避免因未及时归还数据导致阻塞

## 3. 核心模块划分

| 模块 | 职责 | 关键点 |
|------|------|--------|
| **FaceIDPreviewManager** | 核心协调器，统一管理初始化、启动、停止 | 生命周期管理、模块间协调 |
| **CameraManager** | 封装 EvsSDK，管理摄像头枚举、打开/关闭、取流 | 权限检查、Camera ID 解析 |
| **FramePipeline** | 帧处理流水线，三线程协调（取流/处理/渲染） | 阻塞防护、队列管理 |
| **IFaceIDAlgorithm** | 算法抽象接口，定义算法交互协议 | 解耦算法实现 |
| **BufferManager** | 管理 EvsSDK 的 Buffer 生命周期，避免阻塞 | 引用计数、超时回收 |
| **PreviewRenderer** | 将处理后的帧数据渲染到 SurfaceView | 帧同步、性能优化 |

## 4. 算法抽象接口设计 (IFaceIDAlgorithm)

```kotlin
/**
 * Face ID 算法抽象接口
 * 算法团队根据此接口实现具体逻辑
 */
interface IFaceIDAlgorithm {

    /**
     * 算法处理结果
     */
    data class FaceIDResult(
        val faceId: String,              // Face ID 唯一标识
        val confidence: Float,           // 置信度 (0.0 ~ 1.0)
        val faceRect: RectF?,            // 人脸框坐标 (用于画框)
        val processedData: ByteArray,    // 处理后的帧数据（带人脸框绘制）
        val landmarks: List<PointF>?     // 可选：人脸关键点
    )

    /**
     * 初始化算法
     * @param context Android上下文
     * @param config 算法配置参数（模型路径、阈值等）
     * @return 初始化是否成功
     */
    fun initialize(context: Context, config: Map<String, Any> = emptyMap()): Boolean

    /**
     * 处理单帧数据
     * @param frameData 原始帧数据 (YUV / NV21 / RGBA)
     * @param width 图像宽度
     * @param height 图像高度
     * @param format 图像格式标识
     * @return FaceIDResult 包含 faceId + 处理后的帧数据
     */
    fun processFrame(
        frameData: ByteArray,
        width: Int,
        height: Int,
        format: Int
    ): FaceIDResult

    /**
     * 释放算法资源
     */
    fun release()
}
```

## 5. 核心协调层 — FramePipeline 帧处理流水线

这一层是设计的关键，解决**避免因未及时归还数据导致阻塞**的问题。

### 5.1 三线程模型

```
取流线程                 处理线程池                渲染线程
    │                       │                       │
    │──getNextFrame()──▶    │                       │
    │   (获取原始Buffer)     │                       │
    │                       │──processFrame()──▶    │
    │                       │   (算法处理+Face ID)   │
    │                       │                       │──onFrameAvailable()
    │                       │                       │   (渲染到SurfaceView)
    │                       │                       │──doneWithFrame()
    │                       │                       │   (归还Buffer)
    ▼                       ▼                       ▼
```

### 5.2 阻塞防护机制

| 问题 | 解决方案 |
|------|---------|
| 算法处理慢 → Buffer 堆积 | 设置队列最大长度（如 3），超出的帧直接丢弃旧帧 |
| 算法卡死 → Buffer 不归还 | 设置超时机制（如 500ms），超时强制回收 |
| 帧处理 > 帧间隔 | 使用线程池，并行处理；或跳帧处理（每 N 帧处理一次） |

## 6. 调用时序

```
App启动
  │
  ├── 1. IFaceIDAlgorithm.initialize(context, config)
  │         ↓
  ├── 2. CameraManager.openCamera(cameraId)
  │         ↓
  ├── 3. FramePipeline.startCapture()      ← 循环取帧
  │         ↓
  ├── 4. FramePipeline.startProcessing()   ← 算法 processFrame()
  │         ↓                              ← 返回 faceId + processedData
  ├── 5. FramePipeline.startRendering()    ← 显示带框画面
  │         ↓
  └── 6. doneWithFrame()                   ← 归还Buffer

  ... 循环 3~6 ...

关闭时：
  ├── IFaceIDAlgorithm.release()
  ├── CameraManager.closeCamera()
  └── FramePipeline.stop()
```

## 7. 数据流约定（算法团队参考）

```
        原始帧 (YUV/NV21)
               │
               ▼
    ┌─────────────────────┐
    │  自研 Face ID 算法   │
    │                     │
    │  输入: frameData     │
    │       + width       │
    │       + height      │
    │       + format      │
    │                     │
    │  处理:              │
    │  1. 人脸检测         │
    │  2. 特征提取         │
    │  3. 生成 faceId     │
    │  4. 在原帧画人脸框   │
    │                     │
    │  输出: FaceIDResult │
    │  ├── faceId        │  ← 唯一标识
    │  ├── faceRect      │  ← 画框坐标
    │  ├── processedData │  ← 带框帧数据
    │  └── confidence    │  ← 置信度
    └─────────────────────┘
```

## 8. 技术选型建议

| 技术点 | 选项 |
|--------|------|
| **渲染方式** | SurfaceView（性能优先）或 TextureView（支持变换） |
| **帧格式转换** | YUV → RGB（算法输入），使用 RenderScript 或 Native 加速 |
| **线程模型** | 主线程 UI + 子线程取流 + 子线程算法处理 |
| **构建环境** | Android Studio + JDK 17 + Android 10 SDK |
